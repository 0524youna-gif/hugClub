# 허그 클럽 (Hug Club) — 프로젝트 브리프

## 기획 의도
이름도 얼굴도 밝히지 않고 힘든 마음을 꺼내놓을 수 있는 **익명 감성 지지 커뮤니티**. 누군가 글을 남기면 다른 사람들이 "허그(안아주기)" 반응과 사연별 익명 채팅으로 위로를 건넨다. 톤은 가볍고 따뜻하며 과하게 진지하거나 임상적이지 않다. 위로/공감을 직접적으로 강요하지 않는 편안한 어투를 지향한다.

- 사이트 제목: 허그 클럽
- 상단 설명 문구 예시: "오세요 허그클럽. 상처받은 당신을 안아드립니다. (* 주의사항: 공감금지, 연민금지, 이해금지) 사실 해도 돼요"
- 빈 게시판 문구: "여기엔 이름도, 얼굴도 없어요. 하고 싶었던 말을 남겨보세요."

## 기술 스택
- **프론트엔드**: 순수 HTML/CSS/JS 단일 파일 (`index.html`). 프레임워크·빌드 도구 없음.
- **백엔드**: Firebase — Firestore(DB), Firebase Authentication(구글 로그인), Firebase Hosting(배포)
- **프로젝트 ID**: `hugc-d9303`
- **배포 주소**: https://hugc-d9303.web.app
- **저장소**: GitHub `0524youna-gif/hugClub` (main 브랜치가 배포 소스)
- 배포 명령: `firebase deploy --only hosting` (규칙 바꿨으면 `--only hosting,firestore:rules`)

## 핵심 기능
1. **익명 글쓰기** — 로그인 없이 누구나 사연 작성 가능
2. **허그 반응** — 글을 "안아주기". 로그인 후 직접 그린 아이콘(몸+팔)으로 실제 감싸안는 애니메이션 재생, 비로그인이거나 아이콘 없으면 기본 하트+팔 애니메이션
3. **허그 아이콘 그리기** — 얼굴/어깨/팔 3개 레이어로 따로 그리고, 다른 레이어는 반투명 미리보기로 겹쳐 보임. 그린 아이콘은 계속 누적 저장되고, 그중 최대 5개를 "프리셋"으로 지정해서 허그할 때 비리얼(BeReal) 스타일로 골라 쓸 수 있음
4. **댓글(채팅)** — 사연마다 독립적인 익명 닉네임(형용사+동물 조합)과 색이 자동 부여됨. 실시간 채팅형 UI
5. **채팅방 목록** — 내가 메시지를 보낸 사연들을 모아보는 목록
6. **허그 알림** — 내 글에 허그가 오면 실시간으로 알림, 허그한 사람의 아이콘을 눌러 애니메이션 재생 가능 (세션 중에만 유지, 새로고침하면 사라짐)
7. **정렬 필터** — 최신순 / 허그 많은순 / 허그 적은순(외로운 글 띄우기) / 채팅 많은순
8. **글 삭제** — 작성자 본인(브라우저 localStorage 기준 판별) 또는 관리자 계정은 아무 글이나 삭제 가능
9. **공지사항 카드 + 오늘의 질문 카드** — 공지는 고정 문구, 오늘의 질문은 30개 목록에서 날짜 기준으로 매일 자동으로 바뀜
10. **실시간 접속자 수** — "지금 N명이 함께 있어요" (Firestore 하트비트 방식)
11. **배경 클로버 일러스트**, 모바일 허그 시 진동(햅틱) 피드백

## 데이터 모델 (Firestore)
```
notes/{noteId}                    { text, createdAt, hugs, commentCount }
notes/{noteId}/comments/{id}      { name, color, text, ts }        // 사연별 익명 닉네임
notes/{noteId}/hugs/{id}          { uid, iconId, ts }              // 누가 어떤 아이콘으로 안아줬는지
users/{uid}                       { icons: [{id,hugIcon,armsIcon,createdAt}], presetIds: [id,...] }
presence/{visitorId}              { lastSeen }                     // 실시간 접속자 카운트용
```

## 보안 모델
전반적으로 **완전 개방된 익명 신뢰 구조** — 대부분의 write가 `if true`. 실명/계정 기반 서비스가 아니라 익명 게시판이라 서버단에서 "내 글/내 댓글"을 강제할 방법이 없고, 클라이언트(localStorage) 기준으로만 소유권을 판별한다. `users` 컬렉션만 로그인한 본인 소유 문서에만 write 가능(read는 전체 공개 — 허그 알림에서 상대방 아이콘을 보여줘야 해서). 관리자 계정(UID 하드코딩)은 아무 글이나 삭제 가능.
